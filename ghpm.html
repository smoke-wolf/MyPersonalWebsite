Installing

Upon completion of the download, the next step is to extract the contents of the file.

Inside, you will find a directory denoted as "GitHub-Package-Manager." However, it's essential to designate your preferred storage location for this folder before advancing.

Subsequently, in your command line interface, direct yourself to the "GitHub-Package-Manager" directory by employing the 'cd' command.

To initialize the application, execute the 'python3 Start.py' command.

With these actions completed, you are now well-prepared to fully leverage the capabilities of GitHub-Package-Manager.

Usage Of GHPM

Find usage for the GHPM package here.

Install Repositories Through GHPM

  1. Load GHPM:
    • Launch the GHPM application.

  1. Initiate Installation:
    • Within the GUI, select the "Install" option. This selection will prompt a new window.

  1. Specify Repository or Folder:
    • Now, you can automate the installation process between GitHub and local folders.
      v Enter the Git repository you wish to download, or choose a local folder for connection.


  1. Submission:
    • Click the "Submit" button, and you will be presented with a file display.

  1. Automated Installation:
    • The displayed window allows for complex automated installations.
      After selecting a file, the window will close, and the system will automatically manage the required environment for launches, completing the installation.

Activate Connected Applications


Graphical User Interface (GUI):

  • Repo Selection:
    • If you've downloaded a repository from the Installer, it will appear within the "Activator" in the GUI.
    • Standard installations are found in the first layer of the "Activator," which will request you to confirm GitHub.
  • Confirmation:
    • Once you select "Yes" for GitHub, you'll see the following display.
    • If you choose "No," it will prompt for local applications, and another "No" will display complex applications.
  • Application Launch:
    • Select and double-click the option for your application. This action opens a new window reflecting the application as it launches in the chosen format.

Please note that the specific steps for activation may vary depending on the style of the package and the installation method


Uninstall

  • Access Uninstall:
    • From the main menu in the GUI, select the "uninstall" option.


  • Uninstall Menu:
    • This will open a menu that allows you to uninstall both GitHub and Advanced applications.


  • Confirmation:
    • Once you've made your selection, click "Confirm Uninstall." This action will bring up a confirmation window.


  • Uninstall Process:
    • If you select "Yes," the program will attempt to uninstall the application. Depending on how it manages the uninstallation, you will receive a corresponding notification.

    [3] - INSTALL

    1. Begin by specifying the Git repository you wish to access, or alternatively, designate a folder for connection.

If the repository’s readme includes guidance on an entrypoint file (the initial file to execute), please select the indicated entrypoint file.     [4] - ACTIVATE

    Once you have successfully downloaded a repository using the Installer, it will be accessible within the Activator. If it was a repository that was installed it will be under the first layer ‘GitHub’ . If however you connected a local directory you will find it located in the second layer of the Activator interface ‘Local’. Finally, if it was a repository installed through an advanced installation you will find it located in the third layer of the Activator interface ‘Advanced GitHub Applications’.

    GitHub Features:

    After you have downloaded and installed a repository using the GHPM client, there are several features available to optimize your GitHub downloads:

    1. Update: This feature enables you to refresh the repository with the latest version from its source. Updating will not change its save location, but it may have varying effects depending on the functionality of the downloaded repository.

    2. Reinstall: The “Reinstall” option permits users to reconfigure either the entry point file or the build instructions of the installed repository. This can be helpful for modifying how the application runs or is built.

    3. Launch: Use the “Launch” feature to initiate the loaded application. This allows you to run the software or application you’ve downloaded and installed directly from the GHPM client.

    [5] - UNINSTALL

    Uninstalling is a straightforward process that involves selecting the downloaded repository and confirming the uninstall action. This action will effectively remove all files and components associated with the downloaded project, providing a clean and complete removal.

    [6] - COMMAND LINE

    The Command Line offers users encountering bugs access to lower-level functionalities of the project. It is also invaluable for debugging, providing users with comprehensive overviews of logs and events in the backend.

    [7] - UPDATE LOCAAL

    This action enables users to install repositories previously downloaded through their web browser.

    [8] - START SERVER

    Initiate a connection to the browser and accept incoming requests for installations. This feature allows users to conveniently download repositories through their browser’s user interface.

    [9] - KILL SERVER

    Terminate the connection from the GHPM client to the browser by truncating the connection and no longer accepting incoming requests.

    [10] - UPDATE

    Update the GHPM client directly from the user interface. Please note that this feature is still under development. Consider enablingAutomatic Updatesfor an enhanced user experience.

    [11] - GIT LIST

    Effortlessly install any repositories listed; these options come from other GHPM project members or serve as optimizations for the client.

    [12] - GHPM RECOMMENDED

    Engage with a continuously expanding library of repositories and software for experimentation and exploration. This element is currently in active development and holds a high priority. Expect more exciting features in the near future.
